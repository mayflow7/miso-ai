
== instruction 03

1. 말풍선의 생성 x 좌표가 모두 동일한 것 같은데, 각 버튼위 위에서 표시가 시작되도록 변경
   FloatingBubble 버튼별 X 위치 + 애니메이션 개선

2. 말풍선이 너무 빨리 사라지는 것을 화면의 중앙까지 올라가면서 dim out으로 사라지게 변경

 3. 버튼을 누르면 지금은 말풍선 효과밖에 없는데, 좀 더 다양한 이펙트를 넣자. 
    예를 들자면, 백그라운드에 하트 이미지가 fireworks 처럼 터지거나. 여러 귀여운 이모지 이펙트 추가를 검토 
    useBackgroundMusic 훅 생성
    ParticleEffect 컴포넌트 생성 (하트/별 파티클)
    
    
4. D:\_MyService\002_miso.ai\_ref_resource\music를 보면 Dream.wav라는 음악 파일을 앱의 백그라운드 파일로 이용. 
   application 실행하면 백그라운드 음악으로 약간 조용하게 play되게 하고 싶어. 
   Dream.wav → public/audio/ 복사   

5. 오픈쪽 상단에 조그만한 speaker 아이콘을 추가한 후, toggle 방식을 백그라운드 음악을 mute/unmute할 수 있는 기능을 추가

---

6. 백그라운드 음악은 앱이 시작하자마자 자동 플레이 되도록 하고, 자동 loop.

7. 스피커 아이콘은 게이미지에 오버랩되지 않도록 게이지 바 하단에 갭을 두도록 수정. 

8. 스피커 이미지 버튼은 현재 사운드 toggle 기능 밖에 없는데, 사운드 볼류 조절도 가능하도록 기능 추가를

9. 각 버튼 클릭시 popup 되는 메시지를 500개 이상 준비를 해서, 클릭시 랜덤으로 다양한 이미지가 나오도록 수정

---

10. 처음 시작시 LV은 0이고, 클릭하면 클릭할 수도록 level 증가하는데, 현재는 100번 카운트에 level 10 만점으로 세팅되어 있다. 
    레벨 업그레드의 %로 변경하자. 만약 레벨이 0에서 1로 올라가면, 스테이터스 바는 empty부터 시작하고, level 1의 진행 %로 표시 

11. 레벨 업그레이드를 실제 게임과 사용자의 시간이 필요하도록 처음엔 쉽다가 점점 어려워 지는 로직으로 
    예로, 처음 시작은 lv 1로 시작하는데, 30번 클릭하면, 레벨 2으로 업그레이드 되는 점점 시간이 많이 드는 구조. 
        lv 1 -> lv 2는 30번 클릭
        lv 2 -> lv 3는 60번 클릭
        lv 3 -> lv 4는 120 클릭.
        ... 점점 어려워지도록... 

    현재는 맥스멈 lv은 70으로하자. 
    플레이어가 lv 1 -> lv 10 까지 가는데 걸리는 시간은 대략 3일 
    lv 10 -> lv 15 까지는 6일 
    lv 15 -> lv 20 까지는 15일
    lv 20 -> lv 25 까지는 30일 ... 
    등과 같은 구조...
